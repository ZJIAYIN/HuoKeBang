## EchoMind 智能客服系统

**技术栈**：Python、FastAPI、Anthropic Claude API、Redis、Milvus、Prometheus、Docker Compose

**项目描述**：基于 LLM 的企业级智能客服系统，支持三路融合意图识别实现多角色 Agent 路由（文档咨询、技术支持、工单处理），结合 RAG 知识库增强问答与三级记忆管理，实现多场景智能问答与上下文连续对话，并支持人工紧急事件标记与升级

**项目职责：**

- **三路融合意图识别与多角色路由**：设计 LLM + Embedding + Pattern Matching 三路分类体系，通过加权投票融合结果，根据用户意图动态路由至文档咨询、技术支持、工单处理等角色化 Agent；支持紧急事件标记与人工升级

- **RAG 混合检索与在线质量拦截**：基于 Milvus 构建 Dense（BGE-base-zh）+ Sparse（jieba 分词）双向量混合检索架构，通过 Hybrid Search + RRF 融合排序消除分数量纲差异，无需维护双存储一致性；支持 PDF、Word 等多格式文档上传与基于文件 MD5 的精准删除；检索后引入 LLM-as-Judge 置信度评判，低于阈值自动拦截低质量上下文并向 Redis 持久化低分 query，形成反馈闭环指导知识库定向补全

- **三级记忆管理**：设计 Redis 工作记忆 + Milvus 历史摘要记忆 + 用户画像长期记忆三层架构，动态召回历史上下文增强多轮对话连续性；会话超过 15 轮自动触发 LLM 摘要压缩，降低上下文窗口占用与 Token 成本

- **检索评测与全链路可观测**：构建独立检索评测管线 `/eval/retrieval`，LLM-as-Judge 评判 Top-K chunk 与参考答案的语义相关性量化 Recall@K，支持无参考答案时直接评判 chunk 对 query 的可用性，反哺检索策略迭代；集成 Langfuse 对查询改写、并行召回、LLM 重排、生成各阶段进行 Trace 监控，快速定位召回瓶颈
