## EchoMind 留资型智能客服系统

**技术栈**：Python、FastAPI、Anthropic Claude API、ChromaDB、Redis、Docker

**项目描述**：
基于 LLM 构建留资型智能客服系统，通过「意图识别 → RAG问答 → 用户画像更新 → 数据闭环训练」形成完整业务链路，实现专业咨询、留资引导与持续优化。

**职责：**

1、基于 Few-shot Prompt、历史会话摘要及用户画像构建意图识别链路，通过意图+情绪二维矩阵（9意图×5情绪）进行留资时机决策与多角色 Agent 路由，根据用户意图与情绪动态编排咨询、留资、转人工等业务流程。

2、搭建 ChromaDB + BM25 混合检索系统，通过 RRF 融合排序提升知识召回效果；设计递归文本切割策略支持多格式知识库管理。

3、构建用户画像系统，异步抽取并更新车型偏好、预算区间等关键信息，在后续对话中实现个性化回答与留资策略。

4、沉淀 {query, intent, sentiment} 对话数据，建立 Bad Case 标注与回归测试机制，为后续 Qwen2-LoRA 微调本地分类模型积累训练数据，逐步替代线上 Few-shot 分类方案。


1、基于 Few-shot Prompt、历史会话摘要及用户画像构建意图识别链路，一次输出意图与情绪双标签；结合意图-情绪矩阵进行留资时机决策，并驱动咨询答疑、留资确认、转人工等多角色 Agent 路由。

2、构建 Query Rewrite → Hybrid Recall → RRF → Rerank 的 RAG 检索链路；通过向量检索召回语义相关知识、BM25 召回关键业务术语文档，兼顾语义理解与精确匹配能力，并利用融合排序与重排序提升知识命中率；

3、构建用户画像系统，从对话中抽取车型偏好、预算区间等关键信息；采用异步任务更新画像，避免影响主链路响应时延，并在后续对话中实现个性化回答与留资策略。

4、沉淀 {query, intent, sentiment} 对话数据，建立 Bad Case 标注与回归测试机制，为后续本地分类模型训练积累高质量样本，形成“线上收集—人工校正—模型迭代”的数据闭环。
