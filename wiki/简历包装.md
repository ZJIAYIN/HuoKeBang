## 面试高频问题及回答思路

### Q1: 端到端意图识别是怎么做的？为什么用三路融合？

**回答思路**：

单一策略各有短板：
- LLM 准确但延迟高（500-800ms），且有 API 调用成本
- 向量匹配快但对复杂语义理解不够
- 关键词匹配零延迟但太粗糙

所以我设计了三路融合 + 加权投票：
- LLM 作为主力（70%），处理复杂语义和上下文依赖
- Embedding 作为快速通道（20%），对常见表达直接匹配
- Pattern 作为兜底（10%），确保即使 API 故障也有基本能力

三路通过 `asyncio.gather` 并行调用，不增加额外延迟。投票结果低于置信度阈值时降级为 OTHER，避免误判。

**加分点**：提到在线学习——用户纠正后自动更新模板，清除 Embedding 缓存，实现持续改进。

---

### Q2: 检索工具召回不好怎么优化的？

**回答思路**：

两个核心问题对应两个解决方案：

**召回不全** → 查询改写（Query Rewriting）
- 用 LLM 把原始查询改写为 3 个不同角度的子查询
- 例如 "退款流程" → ["如何申请退款", "退款需要多少天", "退款政策"]
- 3 个子查询并行检索，结果合并去重，覆盖面显著提升

**召回不好** → LLM 重排（Reranking）
- 向量相似度高 ≠ 对用户有用（语义鸿沟）
- 用 LLM 对合并结果按"对用户查询的实际帮助程度"重新打分
- 取 Top-K 返回，质量显著优于原始排序

**加分点**：提到完整链路是 `改写 → 并行召回 → 去重 → 重排 → Top-K`，以及熔断器防止检索服务雪崩。

### Q3: 多轮对话的记忆是怎么管理的？

**回答思路**：

三级记忆架构，模拟人类记忆：
- **工作记忆**（Redis）：当前会话最近 20 条，毫秒级读写
- **情景记忆**（ChromaDB）：历史对话的 LLM 摘要，按语义向量检索
- **用户画像**（ChromaDB）：从对话中提炼的偏好和实体

关键机制是**自动压缩**：
- 工作记忆达到 15 条时触发
- LLM 对旧消息生成 2-3 句摘要
- 摘要存入情景记忆，工作记忆只保留最近 5 条
- 这样 context 永远不会爆炸，同时历史信息不丢失

每次请求时三级记忆融合为一个 `MemoryContext`，包含摘要 + 相关历史 + 用户画像 + 最近对话。

**加分点**：提到时效性——Redis 有 24h TTL，情景记忆按语义相关性检索而非时间顺序。

### Q4: 多 Agent 路由是怎么做的？

**回答思路**：

三层路由决策：
1. **意图映射**：根据意图识别结果直接映射到专属 Agent（技术→TechnicalAgent，账单→BillingAgent）
2. **性能路由**：同类 Agent 有多个实例时，按 `routing_score = 成功率 × 0.7 + 低延迟分 × 0.3` 选最优
3. **降级路由**：专属 Agent 失败时自动降级到 GeneralAgent

还支持并行协作：复杂问题（如同时涉及技术和账单）可同时派发给多个 Agent，结果合并返回。

**加分点**：提到 Monitor 闭环——Agent 成功率下降时 routing_score 自动降低，路由权重动态调整，不需要人工干预。

---

### Q5: 如何评测端到端 Agent？

**回答思路**：

用 **LLM-as-Judge** 方案：
- 人工标注成本高、主观性强、不可规模化
- 用 LLM 作为评判者，从相关性、准确性、完整性、有用性四维度打分
- 可重复、可规模化、可自动化

评测流程：
1. 意图识别评测：预测 vs 标注，计算 Accuracy / F1
2. 端到端对话评测：模拟多轮对话，LLM-as-Judge 打分
3. 回归测试：与历史基线对比，退化超 5% 自动标记
4. 自动生成优化建议

**加分点**：提到评测结果可以反馈到在线学习——低分样本自动加入训练集。

---

### Q6: Monitor 是怎么影响路由的？

**回答思路**：

Monitor 和 Orchestrator 形成闭环：
1. 每次请求后，Agent 的 `AgentStats`（成功率、延迟）实时更新
2. Monitor 定时采集（10s），做 Z-score 异常检测
3. 发现异常时触发告警
4. Orchestrator 在选 Agent 时调用 `_best_agent()`，读取 `routing_score()`
5. `routing_score = 成功率 × 0.7 + 低延迟分 × 0.3`
6. 问题 Agent 的 score 自然下降，路由权重自动降低

整个过程不需要人工干预，是自适应的。
