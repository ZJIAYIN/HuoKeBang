技术栈：Python、FastAPI、Anthropic Claude API、Redis、ChromaDB、Prometheus、Docker Compose

项目描述：基于 LLM 构建面向汽车购车场景的留资型智能客服系统，解决传统客服回复效率低、留资率不足的问题。系统通过专业问答建立用户信任，结合多轮对话分析购车意向与用户画像，在合适时机引导用户完成留资，为销售提供高质量潜客线索。

项目职责：

1. 设计 Planner-Orchestrator-ResponseAgent 三层解耦架构。Planner 基于三路融合做意图识别与结构化输出，Orchestrator 统一编排 Skill 和 Tool 调度，ResponseAgent 整合上下文一次生成回复。Planner 针对 JSON 输出做了降级处理，保障线上稳定性。

2. 设计声明式可插拔 Skill 架构。Skill 支持意图路由、语义化版本号、灰度发布，对 user_id 做一致性哈希实现版本路由，按流量比例放量。后台线程定时轮询文件变更，触发热重载，增删改 Skill 无需重启进程。

3. 构建 RAG 检索链路。规则改写 + LLM 多角度改写保证查询多样性，通过向量检索召回语义相关知识、BM25 召回关键业务术语文档，兼顾语义理解与精确匹配能力，并利用融合排序与重排序提升知识命中率。

4. 自研文档切割策略。基于符号优先级做递归分割 + 贪心合并 + overlap(50) 平滑块大小与语义完整性。针对 PDF 表格切块问题，通过特殊标记将表格包裹为原子块，避免表格被拆散。

5. 搭建多意图评测闭环：离线基于 Golden Test Set 统计 TP/FP/FN，以 Macro F1 + Exact Match Rate 量化 Prompt 优化效果；线上嵌入用户反馈埋点，自动收集 badcase 回流至测试集，持续补齐意图识别盲区。
6. 设计 Redis + ChromaDB 三级记忆架构，使用 Redis 管理当前会话工作记忆，使用 ChromaDB 存储历史对话摘要及用户画像，支持摘要压缩、跨会话语义检索及长期记忆管理。
