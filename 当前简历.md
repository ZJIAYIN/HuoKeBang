技术栈：Python、FastAPI、Anthropic Claude API、Redis、ChromaDB、Prometheus、Docker Compose

项目描述：面向汽车购车场景的留资型智能客服系统，支持多意图理解、RAG 问答、长短期记忆管理。

项目职责：

1. 设计 Planner-Orchestrator-ResponseAgent 三层解耦架构。Planner 基于三路融合做意图识别与结构化输出，Orchestrator 统一编排 Skill 和 Tool 调度，ResponseAgent 整合上下文一次生成回复。并针对 Planner 的 JSON 输出做了降级处理，保障线上稳定性。

2. 设计可插拔 Skill 架构。Skill 只声明槽位、工具和指令等元数据，不包含执行逻辑。新增一个 Skill 只需注册一行代码，编排层和生成层无需改动，易于横向扩展。

3. 构建完整 RAG 检索链路。规则改写 + LLM 多角度改写保证查询多样性，向量检索 + 手写 BM25（jieba 分词 + 领域词典）兼顾语义与关键词匹配，RRF 融合 + LLM 重排提升召回质量、避免无效 token 开销。

4. 自研文档切割策略。递归分割 + 贪心合并 + overlap 三阶段切割，平衡块大小与语义完整性。针对 PDF/Word 表格以标记包裹为原子块，避免表格被拆散。

5. 构建评估体系与数据闭环。通过 Golden Test Set 定期回归意图识别效果，线上结果持久化用于 Bad Case 分析，反哺测试集形成持续迭代。
